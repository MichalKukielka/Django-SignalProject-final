The application allows You to use two main modules. The first of them is the Signal Management module. It provides the user with two options. The Input Signal option invokes a form by means of which the user can add his data to the signal database. This form requires data such as signal name, data annotations, signal amplitude unit, sampling frequency, and data file. The second option is to call the list of entered files - Signal List. It allows advanced data management. By this section You could to go to the analysis of a specific file, download result files from the analysis and delete specific records from the database.

The second module is named the Signal Processing. It allows You to select specific data from the list of signals uploaded to the database. The selected signal is displayed and analyzed. It is possible to perform digital filtration on this data. The results of the analysis are saved in the appropriate file.

The user can send data in the following file formats:

  • .mat;
  • .json;
  • .csv.

For .mat and .csv formats, vertical (column) placing of data, without any headers is recommended. The vector of time for data is prepared by the application. While working with the .json format, the data must be under the 'signal_data' key and be a list type.

The analysis results file is prepared in .json format and has the following structure:

{

"main_info": {

"name": "",

"author": "",

"adnotations": "",

"amplitude_unit": "",

"add_date": "",

"last_edit_date": ""

},

"basic_analysis": {

"sample_rate": [],

"signal_values": [],

"fft_values": [],

"frequency_vector": [],

"signal_mean": [],

"signal_power": [],

"signal_energy": []

},

"filration_results": {

"lowpass": {},

"highpass": {},

"bandpass": {}

}

}
